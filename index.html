Validasi Kredensial

Digunakan untuk mengambil data akun dari tabel admin_toko berdasarkan email yang diinputkan pengguna. Limit 1 memastikan hanya 1 record yang diambil untuk performa.

SELECT id_pengguna, nama_pengguna, email, kata_sandi, role
FROM admin_toko
WHERE email = ? 
LIMIT 1;

Update Login Terakhir

Jika login berhasil (password cocok), sistem akan memperbarui waktu login_terakhir menggunakan fungsi waktu server (NOW()).

UPDATE admin_toko 
SET login_terakhir = NOW() 
WHERE id_pengguna = ?;

Laporan Transaksi Bulanan Ekstensif

Query kompleks yang melakukan JOIN antara tabel transaksi, produk, dan admin_toko. Penggunaan logika CASE WHEN secara langsung dalam SQL mempercepat komputasi kalkulasi total harga tergantung apakah barang masuk atau barang keluar. Data difilter berdasarkan bulan dan tahun spesifik.

SELECT
    t.id_transaksi, t.tanggal, t.jumlah, t.status, t.keterangan,
    p.nama_produk, p.satuan, p.kategori, p.harga_beli, p.harga_jual,
    -- Menentukan harga satuan secara dinamis
    CASE WHEN t.status='masuk' THEN p.harga_beli ELSE p.harga_jual END AS harga_satuan,
    -- Mengalkulasi total_harga (jumlah * harga yg sesuai)
    (t.jumlah * CASE WHEN t.status='masuk' THEN p.harga_beli ELSE p.harga_jual END) AS total_harga,
    a.nama_pengguna
FROM transaksi_produk t
JOIN produk p ON p.id_produk = t.id_produk
LEFT JOIN admin_toko a ON a.id_pengguna = t.id_pengguna
WHERE YEAR(t.tanggal) = ? AND MONTH(t.tanggal) = ?
ORDER BY t.tanggal ASC, t.id_transaksi ASC;
DISTINCT

Daftar Bulan Tersedia

Digunakan untuk mendapatkan daftar bulan riwayat transaksi (format YYYY-MM) yang unik (distinct) tanpa terduplikasi. Berfungsi mendaftar opsi pada dropdown laporan maksimal dua tahun terakhir.

SELECT DISTINCT DATE_FORMAT(tanggal, '%Y-%m') AS bulan
FROM transaksi_produk
ORDER BY bulan DESC
LIMIT 24;

Transaksi In / Out

Riwayat Transaksi

Sama seperti query laporan, namun difokuskan untuk di menu history dengan urutan dari yang paling baru diinput (Descending), dengan maksimal penampilan data sebesar 500 line.

SELECT t.*, p.nama_produk, p.satuan, p.gambar, p.kategori, a.nama_pengguna
FROM transaksi_produk t
JOIN produk p ON p.id_produk = t.id_produk
LEFT JOIN admin_toko a ON a.id_pengguna = t.id_pengguna
WHERE t.status = ? -- Opsional jika difilter masuk / keluar saja
ORDER BY t.tanggal DESC, t.created_at DESC 
LIMIT 500;

Table Virtual (DB Views)

Query kompleks pendukung yang sudah dipaketkan ke dalam kerangka Database Layer untuk mempermudah pemanggilan Back-End (`api`). Tersimpan di `db_tokomamasayong.sql`.

VIEW

v_stok_saat_ini

Jantung utama dari sistem stok toko. View ini bertugas menjumlah (SUM) semua histori transaksi masuk dan menguranginya dengan histori transaksi keluar, menghasilkan nilai stok final yg mutlak anti cacat / selisih.

CREATE VIEW `v_stok_saat_ini` AS
SELECT
  p.*,
  COALESCE(SUM(CASE WHEN t.status='masuk'  THEN t.jumlah ELSE 0 END), 0) AS total_masuk,
  COALESCE(SUM(CASE WHEN t.status='keluar' THEN t.jumlah ELSE 0 END), 0) AS total_keluar,
  -- Fungsi GREATEST untuk mencegah minus, kalkulasi Mutlak:
  GREATEST(0,
    COALESCE(SUM(CASE WHEN t.status='masuk'  THEN t.jumlah ELSE 0 END), 0) -
    COALESCE(SUM(CASE WHEN t.status='keluar' THEN t.jumlah ELSE 0 END), 0)
  ) AS stok_saat_ini
FROM `produk` p
LEFT JOIN `transaksi_produk` t ON t.id_produk = p.id_produk
GROUP BY p.id_produk;
VIEW

v_stok_menipis

Data yang diambil dari v_stok_saat_ini diseleksi lebih lanjut, hanya mengambil list di mana total stok saat ini sudah masuk titik kritis kurang dari batas minimal.

CREATE VIEW `v_stok_menipis` AS
SELECT * FROM `v_stok_saat_ini`
WHERE stok_saat_ini < stok_minimum;
VIEW

v_dashboard_stats

View statis yang merangkum 4 query besar menjadi 1 baris (row) saja agar Backend (PHP) dan Frontend (Javascript) tidak keberatan sewaktu memuat halaman paling pertama.

CREATE VIEW `v_dashboard_stats` AS
SELECT
  (SELECT COUNT(*) FROM produk) AS total_produk,
  (SELECT COALESCE(SUM(jumlah),0) FROM transaksi_produk WHERE status='masuk') AS total_masuk,
  (SELECT COALESCE(SUM(jumlah),0) FROM transaksi_produk WHERE status='keluar') AS total_keluar,
  (SELECT COUNT(*) FROM v_stok_menipis) AS stok_menipis;
